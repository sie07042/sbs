1단계: 기본 페이지 구조 만들기

src/pages/Profile.jsx   ← 먼저 빈 페이지 컴포넌트 생성
src/pages/Profile.css   ← 스타일 파일 생성





2단계: 라우터 등록

// src/App.jsx 또는 라우터 설정 파일
<Route path="/profile" element={<Profile />} />






3단계: 폼 UI 구현 (정적)

1. 기본 레이아웃 (GNB, Footer)
2. 폼 필드 구현 (input, select)
3. 버튼 그룹 (저장, 취소)
4. CSS 스타일링





4단계: 상태 관리 추가

1. useState로 formData 상태 정의
2. handleChange 함수로 입력값 반영
3. 유효성 검사 로직 (validateForm)
4. errors 상태로 에러 메시지 표시






5단계: 이미지 업로드 기능

1. useRef로 숨겨진 input 참조
2. FileReader로 미리보기 생성
3. 파일 유효성 검사 (타입, 크기)
4. FormData로 서버 업로드






6단계: API 연동

1. 프로필 조회 (GET /api/user/profile) - useEffect
2. 이미지 업로드 (POST /api/upload/image)
3. 프로필 수정 (PUT /api/user/profile)
4. 에러 처리 및 사용자 피드백






7단계: 리팩토링 (선택)

1. 커스텀 훅 분리 (useProfileForm)
2. 하위 컴포넌트 분리 (ProfileImageSection)
3. 재사용 컴포넌트 (FormField)






파일별 구현 순서 (현재 구조 기준)
순서	파일	설명
1	Profile.css	스타일 정의
2	useProfileForm.js	상태 + API 로직
3	ProfileImageSection.jsx	이미지 UI
4	Profile.jsx	메인 컴포넌트 조립







핵심 개념 학습 순서

1. useState     → 폼 데이터 관리
2. useEffect    → 초기 데이터 로드
3. useRef       → DOM 요소 참조 (파일 input)
4. FileReader   → 이미지 미리보기
5. FormData     → 파일 업로드
6. axios        → API 통신
7. Custom Hook  → 로직 재사용
